# Brainstorm: Menu Grass Field UX/Gamification

You are brainstorming a creative community feature for a Korean nutrition calculator app (subwayyy.kr). The app targets Korean dieters in their 20s-30s, covering Subway, Salady, and Poke All Day franchises. The app is mobile-first, clean/minimal design (Toss/Baemin style).

## The User's Idea
A "메뉴 잔디밭" (Menu Grass Field) feature inspired by GitHub's contribution graph:
- Users share/recommend their menu combinations → a grass square gets planted on a shared field
- Minecraft-style aesthetic: dirt/grass blocks as ground
- Random Minecraft-style pixel cats sitting on grass blocks
- Tapping a cat shows a speech bubble with what menu someone ate/recommended

## Existing Design Additions
- Cats colored by brand (green=Subway, purple=Salady, blue=Poke)
- Grass color intensity varies by calorie (light green=low cal, dark green=high cal) — like GitHub's commit intensity
- GitHub contribution graph grid layout

## What I Need
Give me your TOP 5-7 creative, concrete, ACTIONABLE ideas. For each idea:
- Name it
- Describe the UX flow (user does X → sees Y)
- Why it works for Korean 20-30대 dieters
- Implementation complexity (low/medium/high)

Think about:
1. Gamification hooks — what makes users return to "plant" their menus?
2. Social dynamics — how to feel alive with few users initially?
3. Practical value — beyond cute, how does this help dieters?
4. Micro-interactions — delightful animations, visual feedback?
5. Alternative metaphors — is grass+cats the best? Would garden, aquarium, pixel town, etc. work better?
6. Risks — spam, empty field with few users, inappropriate content?

This is a solo developer's side project. Be creative but keep scope realistic.
