# subwayyy 고도화 실행계획 (트래픽/재방문/광고수익)

## 1) 현재 진단 요약

- 기본 SEO/광고 토대는 이미 존재
  - 메타/OG/JSON-LD/GA/AdSense: `templates/base.html`, `templates/index.html`, `templates/salady.html`, `templates/poke.html`
  - 사이트 운영 파일: `public/sitemap.xml`, `public/robots.txt`, `public/ads.txt`
- 하지만 성장 병목이 존재
  - canonical 불일치(일부 외부 canonical)
  - 실제 라우트와 sitemap 정합 이슈 가능
  - placeholder canonical 존재(`templates/pdf_to_text.html`)
  - 계산기 중심 구조로 롱테일 랜딩 수 부족
  - 재방문 유도 기능(저장/히스토리/추천) 약함

## 2) 목표

- 검색 유입 확대: 롱테일 검색 점유율 강화
- 재방문 증대: 계산 이후 사용자 루프 설계
- 광고수익 개선: UX 훼손 없이 RPM 상승

## 3) 90일 로드맵

### Phase 1 (0-2주): 기술 SEO 안정화

1. canonical 정책 통일(도메인/경로 일관성)
2. sitemap와 실제 라우트 정합성 점검 및 수정
3. noindex 대상 페이지 정책 정리
4. Google Search Console / Naver Search Advisor 점검 루틴 구축

### Phase 2 (3-6주): 프로그램형 SEO 확장

1. 랜딩 페이지 확장: `브랜드 x 메뉴 x 목표(저칼로리/고단백/저염)`
2. FAQ/QAPage/Breadcrumb 구조화 데이터 확장
3. 검색 의도형 콘텐츠 묶음 구성
   - 예: "서브웨이 다이어트 조합", "샐러디 저염 메뉴", "포케올데이 고단백 조합"

### Phase 3 (7-10주): 재방문 기능 구축

1. 최근 조합 저장
2. 목표 칼로리 대비 비교 기능
3. 주간 리포트/요약
4. 공유 카드(인스타/카톡) CTA를 계산 완료 직후 배치

### Phase 4 (11-12주): 수익화 최적화

1. CLS-safe 광고 슬롯(레이아웃 선점)
2. Auto ads + 수동 슬롯 병행 A/B
3. 모바일 anchor/vignette 제한 테스트
4. UX 저해 없는 광고 밀도 조절

## 4) KPI

- 검색 성과: 노출수, 클릭수, CTR, 롱테일 유입 비중
- 행동 지표: 세션당 페이지뷰, 평균 체류시간, 7일/30일 재방문율
- 수익 지표: RPM, 광고 viewability, 세션당 수익
- 기술 지표: Core Web Vitals (CLS < 0.1 우선)

### 3개월 목표(가이드)

- 유기 트래픽 2배
- 재방문율 +30%
- RPM +20~40%

## 5) 즉시 실행 우선순위 (Top 4)

1. 기술 SEO 결함 우선 수정(canonical/라우트/sitemap 정합)
2. 메뉴 상세 랜딩 30개 선출시
3. 계산 결과 화면에 저장+공유+추천 CTA 3종 추가
4. 광고 배치 실험을 2주 단위로 운영하고 성과 비교

## 6) 운영 원칙

- 검색 노출만이 아니라 재방문 루프를 같이 설계한다.
- 광고 최적화는 정책 준수와 UX 보호를 전제로 한다.
- 변경은 항상 KPI와 함께 실험(A/B)하고, 승자안만 유지한다.
